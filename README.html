听瓜

一个通过西瓜敲击声采集、人工标注、训练 MFCC + 小 MLP，并把 TFLite 模型包部署回手机端离线推理的 Android 原型。

录音

使用手机麦克风录制 16 kHz 单声道 WAV，每条样本最长 3 秒。

特征

App 提取 8 个基础声学特征，加 13 维 MFCC 均值和 13 维 MFCC 标准差，共 34 维。

标注

用户选择偏生、成熟、过熟/异常，App 把音频和标签写入 metadata.jsonl。

部署

训练脚本输出 model_bundle.zip，App 上传后优先使用 model.tflite 本地推理。

构建 APK

.\gradlew.bat assembleDebug

产物位于 app/build/outputs/apk/debug/SelWatermelon-debug-1.0.apk。

采集流程

  1. 安装 APK，并授予录音权限。
  2. 点击“开始录音”，靠近西瓜敲击 1-3 次，再点击停止，或等待 3 秒自动停止。
  3. 查看判断结果和 MFCC/声学特征。
  4. 切开或用可靠方法确认真实成熟度后，选择偏生、成熟、过熟/异常，并保存反馈。
  5. 采集足够样本后，点击“打包下载数据”导出 watermelon_dataset.zip。

训练 TFLite

pip install tensorflow
python training/train_model.py watermelon_dataset.zip -o model_bundle.zip

脚本会训练一个小 MLP：Dense 128 -> Dropout -> Dense 64 -> Softmax。输入是和 App 端一致的 34 维特征。

回传模型

  1. 把 model_bundle.zip 发送到手机。
  2. 在 App 中点击“上传训练好的 TFLite 模型包”。
  3. 选择 model_bundle.zip。后续录音会优先使用 MLP TFLite。

注意

  • 每个西瓜建议多位置、多次敲击，最终用多次结果综合判断。
  • 训练和测试时应按“西瓜个体”划分，避免同一个西瓜的不同敲击同时出现在训练集和测试集。
  • 真实标签最好来自切开验证、糖度或明确口感反馈，只靠听感标注会污染模型。
