听瓜

一款 Android 西瓜成熟度辅助判断应用。它通过拍打录音、实时波形、声学特征和本地模型推理，给出偏生、尚可、甘甜、过熟等判断，并把真实口感反馈沉淀为可训练数据。

v1.1.0 Android Java 16 kHz WAV MFCC 特征 TFLite / JSON 模型 离线推理

1.1.0 更新重点

甘甜独立判断

“甘甜”现在是尚可和过熟之间、更偏过熟的最佳甜度区间，不再只作为人工反馈标签存在。

外部模型适配

外部模型没有 bursting 标签时，应用会用声学特征后处理适配“甘甜”结果。

列表操作确认

删除前增加确认弹框；下载所选前增加提示弹框，降低误操作风险。

下载后清理

下载弹框默认勾选“下载后删除有反馈数据”，只清理已反馈记录，未反馈数据保留。

功能概览

挑瓜录音

使用手机麦克风采集 16 kHz 单声道 WAV，适合在西瓜侧边拍打 1-3 次完成一次样本采集。

AI 判断

提取 RMS、峰值、过零率、频谱质心、频段比例、衰减和 MFCC 等特征，输出成熟度标签与置信度。

反馈采集

记录实际口感和重量。保存后的反馈会写入导出数据，便于持续训练。

模型评估

可上传多个 model.json 或 model_bundle.zip，在应用内切换并对比不同模型效果。

使用流程

  1. 安装 APK，并授予录音权限。
  2. 在“挑瓜”页点击“开始录音”。
  3. 按提示在西瓜侧边拍打 1-3 次，停止后查看 AI 判断结果。
  4. 切开或确认口感后，选择“偏生 / 尚可 / 甘甜 / 过熟”，可选填写重量。
  5. 点击“保存实际口感”，应用会保存反馈并用于后续训练。
  6. 在“列表”页查看、播放、修改、删除或多选下载录音数据。

成熟度标签

偏生

unripe。高频比例或频谱重心偏高，可能还不够成熟。

尚可

ripe 或 acceptable。成熟度可接受，声音低沉、共振较集中。

甘甜

bursting。位于尚可和过熟之间，更接近过熟，代表最佳甜度区间。

过熟

overripe。低频或衰减特征偏强，可能偏软、空心或过熟。

构建 APK

项目使用 Gradle Android 插件构建，Java 版本为 17。

$env:JAVA_HOME='D:\android-studio\jbr'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
.\gradlew.bat assembleDebug

Debug APK 输出位置：

app/build/outputs/apk/debug/SelWatermelon-debug-1.1.0.apk

模型训练

默认训练线性 JSON 模型，适合小样本快速迭代。

python training/train_model.py watermelon_dataset.zip -o model.json

也可以训练 TFLite 模型包：

python training/train_model.py watermelon_dataset.zip --method mlp -o model_bundle.zip

导出数据

导出的 metadata.jsonl 每行对应一条录音，包含音频路径、采样时间、手机型号、模型判断、用户反馈、重量和声音特征。

样本信息

audio_id、file、created_at_text、duration_ms

设备信息

phone_manufacturer、phone_model、phone_device、phone_label

判断与反馈

model_label、model_confidence、feedback、weight_kg

声音特征

rms、peak、zcr、centroid、MFCC 统计值等

采集建议

  • 每个西瓜建议采集多个位置、多个敲击样本。
  • 训练集和测试集应按“西瓜个体”划分，避免同一个西瓜的不同录音同时进入训练和测试。
  • 真实标签最好来自切开验证、糖度或明确口感反馈，只靠听感标注会影响模型质量。
  • 不同手机麦克风差异明显，导出数据中会记录手机型号，训练时可用于分析设备偏差。

这是辅助判断工具，不替代人工经验或真实口感验证。模型效果取决于样本数量、标签质量和采集环境一致性。
