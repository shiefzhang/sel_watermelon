听瓜版本更新历史

记录 Android 应用、模型判断逻辑、训练脚本和数据管理流程的主要变化。

1.1.0

当前版本 2026-05-30

成熟度判断

  • 新增“甘甜”作为独立判断阶段，位于“尚可”和“过熟”之间，并更偏向“过熟”，用于表达最佳甜度区间。
  • 内置 JSON 质心模型增加 bursting 标签，完整标签为 unripe / ripe / bursting / overripe。
  • 外部模型没有 bursting 标签时，应用会根据声学特征在 ripe / acceptable / overripe 结果附近适配出“甘甜”。
  • “过熟”判断收窄为更明显的偏软、空心或异常区间，避免把最佳甜度区间整体归为过熟。

列表与数据管理

  • 单条删除和批量删除增加确认弹框，降低误删风险。
  • 批量下载增加确认弹框。
  • 下载确认弹框中新增“下载后删除选中数据中有反馈的数据”复选框，默认勾选。
  • 勾选后，下载完成会删除选中数据中已经填写反馈的记录，未填写反馈的数据会保留。

模型管理与训练

  • 支持上传、切换和删除训练好的模型包。
  • 训练脚本支持默认导出线性 JSON 模型，适合小样本快速迭代。
  • 训练脚本保留 MLP TFLite 导出能力，可通过 --method mlp 使用。
  • 训练脚本支持优先使用导出元数据中的特征列，并输出交叉验证报告。

1.0.0

初始版本 2026-05-23
  • 初始 Android 应用。
  • 支持拍打录音、实时波形、声学特征提取和本地成熟度判断。
  • 支持保存实际口感、重量和音频元数据。
  • 支持导出 WAV 样本和 metadata.jsonl。
  • 提供基础训练脚本，用于生成可上传的模型包。
